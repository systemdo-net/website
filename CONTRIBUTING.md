# 参与贡献

感谢您对 SystemDo 的关注与贡献！🎉

## 🤝 如何参与

### 报告问题

- 使用 GitHub Issues 提交问题
- 清晰描述问题并提供复现步骤
- 附上相关系统信息和截图

### 功能建议

- 提交带 "enhancement" 标签的 Issue
- 描述功能用途和对用户的价值
- 在编码实现前先讨论方案思路

### 提交合并请求（Pull Request）

1. Fork 本仓库
2. 创建功能分支（`git checkout -b feature/新功能`）
3. 提交修改（`git commit -m '添加新功能'`）
4. 推送到分支（`git push origin feature/新功能`）
5. 创建 Pull Request

### 代码风格

- 遵循项目已有的代码风格和规范
- 编写清晰的 Commit Message
- 尽可能包含测试
- 同步更新相关文档

## 📋 开发流程

1. 从 Issue 中认领任务
2. 讨论解决方案思路
3. 编码实现并编写测试
4. 提交代码待审核
5. 根据反馈修改
